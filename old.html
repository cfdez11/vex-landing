Features

Everything you need, nothing you don't

Built to stay out of your way. Vex gives you powerful primitives without magic.

File-based Routing

Drop a page.html anywhere in pages/ and it becomes a route. Dynamic segments with [param] folders.

Vue-like Reactivity

Familiar reactive(), computed(), effect(), and watch() primitives built right in, without a virtual DOM.

SSR, CSR, SSG & ISR

Choose the rendering strategy per page. Server-Side, Client-Side, Static Generation, and Incremental Static Regeneration.

Nested Layouts

Root and custom layouts with layout.html. Compose structure without repeating yourself.

SPA Navigation

Client-side routing with no full reloads. Automatic link prefetching on hover for instant page transitions.

Streaming & Suspense

Progressive loading with fallback UI. Streams HTML as it's ready, keeping your app feeling fast from the first byte.

Template Syntax

HTML-first,
nothing to compile.

Write components in plain .html files with familiar directives. No JSX, no build step for templates.

Directives: v-if, v-for, @click, :attr
Interpolation: Double curly brace {"{{ value }}"} syntax
Server & Client components with scoped scripts and styles
Tailwind CSS integrated out of the box
pages/users/page.html 
<!-- server script -->
<script server>
export async function load() {
  const users = await fetchUsers()
  return { users }
}
</script>

<!-- template -->
<ul class="user-list">
  <li
    v-for="user in users"
    :key="user.id"
  >
    {{ user.name }}
  </li>
</ul>

<!-- reactive client logic -->
<script>
const count = reactive(0)
</script>

Rendering

Your strategy, per page

Pick the right rendering mode for each route. Mix and match freely.

SSR

Server-Side Rendering

HTML rendered on every request. Great for dynamic, user-specific content. Optimal for SEO with real-time data.

CSR

Client-Side Rendering

Rendering happens in the browser. Ideal for highly interactive pages with reactive data that updates frequently.

SSG

Static Site Generation

Pages pre-built at compile time. Maximum performance, deploy anywhere. Perfect for docs, blogs, and marketing pages.

ISR

Incremental Static Regen.

Static pages that refresh after a set interval. Get the speed of static with the freshness of server rendering.

Quick Start

Up in 60 seconds

From zero to a running Vex app in three commands.

1

Install the package

npm install @cfdez11/vex
2

Create your first page at pages/page.html

pages/page.html 
<h1>Hello from Vex!</h1>

<script>
const message = reactive('Hello, Vex!')
</script>
3

Start the dev server

node --watch server.js

Visit localhost:3000 and your page is live. Vex auto-discovers all page.html files — no route registration needed.

Start building with Vex

No boilerplate. No config files. Just files, routes, and reactivity.
